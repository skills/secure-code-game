🏗️ Terraform Documentation

Last updated: February 2026 · v1.8

Terraform enables you to safely and predictably create, change, and improve infrastructure. It codifies APIs into declarative configuration files that can be shared and versioned.

Core Concepts

  • Providers: Plugins that interact with cloud APIs (AWS, Azure, GCP)
  • Resources: Infrastructure components (VMs, networks, databases)
  • State: The mapping between your config and real-world resources
  • Modules: Reusable, shareable packages of Terraform configuration

Getting Started

# main.tf — Define your infrastructure
terraform {
  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }
  }
}

provider "aws" {
  region = "us-east-1"
}

resource "aws_instance" "prodbot" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t3.medium"

  tags = {
    Name        = "prodbot-api"
    Environment = "production"
  }
}

Workflow

  1. terraform init — Initialise the working directory
  2. terraform plan — Preview changes before applying
  3. terraform apply — Apply the configuration
  4. terraform destroy — Tear down managed infrastructure

State Management

Store state remotely to enable team collaboration and prevent conflicts:

terraform {
  backend "s3" {
    bucket         = "prodbot-terraform-state"
    key            = "prod/terraform.tfstate"
    region         = "us-east-1"
    encrypt        = true
    dynamodb_table = "terraform-locks"
  }
}

Best Practices

  • Always run terraform plan before apply
  • Use remote state with locking to prevent concurrent modifications
  • Pin provider versions to avoid unexpected breaking changes
  • Use modules for reusable, tested infrastructure patterns
  • Never hardcode secrets — use environment variables or vault
  • Enable drift detection to catch manual changes

Security Considerations

  • Encrypt state files at rest (sensitive data may be stored in state)
  • Restrict access to state backends with IAM policies
  • Use sensitive = true for variables containing secrets
  • Scan configurations with tfsec or checkov
