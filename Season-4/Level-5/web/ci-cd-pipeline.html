🚀 CI/CD Pipeline Best Practices

2026 Edition · Updated

A well-designed CI/CD pipeline is the backbone of modern software delivery. This guide covers the essential patterns and anti-patterns for building reliable, secure, and fast pipelines.

Pipeline Architecture

A production-grade pipeline should include these stages:

  1. Source: Git push triggers the pipeline
  2. Build: Compile code, build container images
  3. Test: Unit tests, integration tests, security scans
  4. Deploy to Staging: Automatic deployment to staging environment
  5. Approval Gate: Manual or automated approval
  6. Deploy to Production: Canary or blue-green deployment
  7. Verify: Smoke tests and health checks

GitHub Actions Example

name: CI/CD Pipeline
on:
  push:
    branches: [main]
  pull_request:

jobs:
  build-and-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
      - run: npm ci
      - run: npm test
      - run: npm run build

  security-scan:
    needs: build-and-test
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npx audit-ci --critical

  deploy-staging:
    needs: security-scan
    runs-on: ubuntu-latest
    environment: staging
    steps:
      - run: echo "Deploying to staging..."

  deploy-production:
    needs: deploy-staging
    runs-on: ubuntu-latest
    environment: production
    steps:
      - run: echo "Deploying to production..."

Security in CI/CD

Pipelines are a high-value target for supply chain attacks. Secure them by:

  • Pinning action versions with SHA hashes, not tags
  • Using OIDC for cloud authentication (no long-lived secrets)
  • Enabling branch protection and required reviews
  • Scanning dependencies with Dependabot or Renovate
  • Signing container images with Sigstore/Cosign
  • Using ephemeral, hardened runners

Common Anti-Patterns

  • ❌ Storing secrets as plain text in pipeline configs
  • ❌ Running tests in production environments
  • ❌ Skipping security scans to "ship faster"
  • ❌ Using latest tags for base images
  • ❌ No rollback strategy for failed deployments

Performance Tips

  • Cache dependencies between runs (actions/cache)
  • Parallelise independent jobs
  • Use build matrices for multi-platform testing
  • Set up incremental builds for monorepos
  • Use self-hosted runners for heavy workloads
